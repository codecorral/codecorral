## ADDED Requirements

### Requirement: Deterministic session title generation
The engine SHALL generate session titles using the pattern `cc-{shortId}-{phase}` where `shortId` is the random suffix portion of the instance ID and `phase` is a workflow-provided phase tag.

#### Scenario: Title generation from instance ID and phase
- **WHEN** `generateSessionTitle` is called with instance ID `test-v0.2-abc12345` and phase `setup`
- **THEN** the returned title is `cc-abc12345-setup`

#### Scenario: Title generation with longer definition prefix
- **WHEN** `generateSessionTitle` is called with instance ID `unit-v0.3-xy9z4w2q` and phase `elab`
- **THEN** the returned title is `cc-xy9z4w2q-elab`

### Requirement: Title sanitization
The engine SHALL sanitize session titles to contain only characters matching `[a-z0-9-]`. Any other characters SHALL be replaced with a hyphen. Consecutive hyphens SHALL be collapsed to a single hyphen. Leading and trailing hyphens SHALL be removed.

#### Scenario: Uppercase characters lowered
- **WHEN** a title component contains `MyPhase`
- **THEN** it is sanitized to `myphase`

#### Scenario: Special characters replaced
- **WHEN** a title component contains `hello_world.test`
- **THEN** it is sanitized to `hello-world-test`

#### Scenario: Consecutive hyphens collapsed
- **WHEN** sanitization produces `cc--abc--setup`
- **THEN** the result is `cc-abc-setup`

### Requirement: Title length enforcement
Session titles SHALL NOT exceed 60 characters. If the generated title exceeds 60 characters, the `shortId` portion SHALL be truncated from the right to fit within the limit while preserving the `cc-` prefix and `-{phase}` suffix.

#### Scenario: Title within limit
- **WHEN** the generated title is `cc-abc12345-setup` (17 characters)
- **THEN** the title is returned unchanged

#### Scenario: Title exceeding limit
- **WHEN** the generated title would be 65 characters
- **THEN** the `shortId` portion is truncated so the total title is exactly 60 characters

### Requirement: Session discovery by title prefix
The engine SHALL provide a function to derive a session discovery prefix from a workflow instance ID, returning `cc-{shortId}` which can be used to find all sessions belonging to that workflow instance.

#### Scenario: Discovery prefix generation
- **WHEN** `getSessionPrefix` is called with instance ID `test-v0.2-abc12345`
- **THEN** the returned prefix is `cc-abc12345`

#### Scenario: Prefix used for filtering
- **WHEN** the prefix `cc-abc12345` is used to filter a session list containing titles `cc-abc12345-setup`, `cc-abc12345-agent`, and `cc-xyz99999-setup`
- **THEN** only the two `cc-abc12345-*` sessions are matched

### Requirement: Short ID extraction from instance ID
The engine SHALL extract the short ID by taking the portion of the instance ID after the last hyphen-separated segment that constitutes the random suffix (the 8-character random string generated by `generateInstanceId`).

#### Scenario: Short ID extraction
- **WHEN** `extractShortId` is called with instance ID `test-v0.2-abc12345`
- **THEN** the returned short ID is `abc12345`

#### Scenario: Short ID from definition with hyphens
- **WHEN** `extractShortId` is called with instance ID `my-workflow-v1-xyz99999`
- **THEN** the returned short ID is `xyz99999` (last 8 characters of the ID)
